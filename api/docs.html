Authentication without the Tax.

Ageinx handles your entire login and signup flow. Unlimited users. Zero lock-in. Built for scale.

Why Ageinx?

Most auth providers charge a "Monthly Active User" (MAU) tax. They punish you for growing. Ageinx operates on a flat tier system so you can scale to millions of users without your bill exploding.

We provide a hosted authentication page. You simply redirect users to us, we authenticate them securely, and redirect them back to your app with a signed JWT.

Quickstart Flow

Integrate Ageinx into any tech stack in 3 simple steps.

Step 1: The Redirect

Direct your users to your dedicated Ageinx hosted auth page. Replace {your-slug} with the slug found in your dashboard.

# Redirect user to this URL when they click "Login"
https://ageinx.vercel.app/auth/{your-slug}

Step 2: The Callback

After a successful login or signup, Ageinx redirects the user back to the callback_url you specified in your dashboard. The secure access token is appended as a URL hash.

// The user lands back on your app here:
https://yourapp.com/dashboard#token=eyJhbGciOiJIUzI1Ni...

Step 3: Verify the Token

Send that token to your backend. Your backend then verifies the token with Ageinx to ensure it is authentic and hasn't expired.

GET https://ageinx.vercel.app/auth/userinfo
Authorization: Bearer <token>

Vanilla JavaScript

Handling the Ageinx callback using pure HTML and JavaScript.

Parsing the Hash

When the user lands on your callback URL, extract the token from the hash, clear it from the URL bar (for security), and save it to localStorage.

// 1. Extract token from the URL hash
const hash = window.location.hash.substring(1);
const params = new URLSearchParams(hash);
const token = params.get('token');

if (token) {
    // 2. Save token for future API requests
    localStorage.setItem('ax_user_token', token);
    
    // 3. Clean the URL so the token isn't visible
    window.history.replaceState(null, null, window.location.pathname);
    
    alert("Login Successful!");
}

React / Next.js

A clean useEffect hook to handle the Ageinx redirect in modern frontend frameworks.

The Auth Hook

Place this logic in the component that acts as your callback_url.

import { useEffect, useState } from 'react';
import { useRouter } from 'next/router';

export default function AuthCallback() {
    const router = useRouter();
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        const hash = window.location.hash.substring(1);
        const token = new URLSearchParams(hash).get('token');

        if (token) {
            localStorage.setItem('ax_user_token', token);
            // Redirect to protected dashboard
            router.push('/dashboard'); 
        } else {
            router.push('/login');
        }
    }, []);

    return <div>Authenticating...</div>;
}

Node.js / Express

Verify the user's token securely on your Node backend.

Verification Middleware

Use this middleware to protect your API routes. It calls Ageinx to ensure the token is legitimate.

const axios = require('axios');

async function requireAuth(req, res, next) {
    const token = req.headers.authorization?.split(' ')[1];
    
    if (!token) return res.status(401).json({ error: "No token" });

    try {
        // Verify with Ageinx Server
        const response = await axios.get('https://ageinx.vercel.app/auth/userinfo', {
            headers: { 'Authorization': `Bearer ${token}` }
        });
        
        // Attach secure user data to the request
        req.user = response.data; 
        next();
    } catch (error) {
        res.status(401).json({ error: "Invalid or expired token" });
    }
}

Python / FastAPI

Verify tokens at lightspeed using FastAPI and HTTPX.

Dependencies Injection

Create a dependency that automatically verifies the bearer token on protected routes.

import httpx
from fastapi import FastAPI, Depends, HTTPException
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials

app = FastAPI()
security = HTTPBearer()

async def verify_user(credentials: HTTPAuthorizationCredentials = Depends(security)):
    async with httpx.AsyncClient() as client:
        response = await client.get(
            "https://ageinx.vercel.app/auth/userinfo",
            headers={"Authorization": f"Bearer {credentials.credentials}"}
        )
        
        if response.status_code != 200:
            raise HTTPException(status_code=401, detail="Invalid token")
            
        return response.json()

# Protect any route easily:
@app.get("/api/protected")
async def protected_data(user: dict = Depends(verify_user)):
    return {"message": "Success", "user_email": user["email"]}
