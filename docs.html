5-Minute Setup

Just three standard HTTP steps to integrate secure JWT authentication.

Requirement: Auth only works on HTTPS domains (except for localhost environments during development).

1. The Redirect

Direct users to your hosted Ageinx page. Replace {your-slug} with your project ID.

# Attach to your "Login" button
GET https://ageinx.vercel.app/auth/{your-slug}

2. The Callback

Ageinx securely authenticates the user and redirects back to your app.

# User lands here after successful login
https://yourapp.com/dashboard#token=eyJhbGciOiJIUzI...

3. Backend Verification

Never trust the frontend. Send the token to your backend to verify it with Ageinx.

GET https://ageinx.vercel.app/auth/userinfo
Authorization: Bearer <token>

Vanilla JavaScript

const hash = window.location.hash.substring(1);
const token = new URLSearchParams(hash).get('token');

if (token) {
    localStorage.setItem('ax_token', token);
    // CRITICAL: Remove token from URL immediately
    window.history.replaceState(null, null, window.location.pathname);
    window.location.href = '/dashboard';
}

React / Next.js

import { useEffect } from 'react';
import { useRouter } from 'next/router';

export default function AuthCallback() {
    const router = useRouter();

    useEffect(() => {
        const token = new URLSearchParams(window.location.hash.substring(1)).get('token');
        if (token) {
            localStorage.setItem('ax_token', token);
            router.push('/dashboard'); 
        } else {
            router.push('/login');
        }
    }, []);

    return <div>Authenticating...</div>;
}

Node.js / Express

Backend verification is mandatory.

const axios = require('axios');

async function requireAuth(req, res, next) {
    const token = req.headers.authorization?.split(' ')[1];
    if (!token) return res.status(401).json({ error: "Denied" });

    try {
        // OPTIMIZATION: Cache /userinfo response (5-10 mins) using Redis to reduce latency.
        // Avoid calling Ageinx on every single request without caching.
        const { data } = await axios.get('https://ageinx.vercel.app/auth/userinfo', {
            headers: { 'Authorization': `Bearer ${token}` }
        });
        
        req.user = data; 
        next();
    } catch (error) {
        // SECURITY: Fail closed on verification failure (deny access)
        return res.status(401).json({ error: "Invalid or expired token" });
    }
}

Python / FastAPI

import httpx
from fastapi import FastAPI, Depends, HTTPException
from fastapi.security import HTTPBearer

app = FastAPI()
security = HTTPBearer()

async def verify_user(credentials = Depends(security)):
    async with httpx.AsyncClient() as client:
        response = await client.get(
            "https://ageinx.vercel.app/auth/userinfo",
            headers={"Authorization": f"Bearer {credentials.credentials}"}
        )
        if response.status_code != 200:
            raise HTTPException(status_code=401, detail="Token invalid")
        return response.json()

Security & Trust

Built-in protections to ensure your credentials are ironclad.

  • No Plaintext Storage: Ageinx never stores plaintext credentials. Passwords are heavily hashed and salted (bcrypt/argon2).
  • Absolute Isolation: We never expose user passwords to your app. Only the user ID and Email are returned from the /userinfo endpoint.
  • Token Lifecycle: Tokens expire in 60 minutes (short-lived for security). They are stateless JWTs signed using HS256/RS256 algorithms.
  • Attack Prevention: Replay attack protection is handled entirely server-side.

Common Mistakes to Avoid

Don't fall into these common implementation traps.

  • ❌ Trusting frontend data: Checking for a token on the frontend is UX, not security. Backend verification is mandatory.
  • ❌ Not clearing the URL hash: Failing to run history.replaceState means users might copy their URL and accidentally send their active token to someone else.
  • ❌ Ignoring 401 errors: Tokens expire. Your frontend must catch 401s from your API and seamlessly redirect the user back to the login route.
  • ❌ Insecure storage: Using localStorage is fine for side projects, but for enterprise apps, transfer the token to an HTTP-Only cookie via your backend.
